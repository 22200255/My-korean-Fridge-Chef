# 👨‍🍳 My Korean Chef (냉장고 파먹기)

"냉장고에 있는 재료, 더 이상 썩히지 마세요!"
My Korean Chef는 사용자가 보유한 식재료를 기반으로 한국 요리 레시피를 검색하고, 나만의 레시피북을 만들어 관리할 수 있는 SPA(Single Page Application) 서비스입니다.

# 🚀 프로젝트 개요

## 프로젝트명: My Korean Chef

### 주요 기능: 재료 기반 레시피 검색, 알레르기 정보 제공, 나만의 레시피북 관리 (CRUD)

사용 기술:

- Frontend: React (Hooks, Router, Context API)

- Style: React-Bootstrap

- API: 식품의약품안전처 조리식품의 레시피 DB

- State Management: Context API + useReducer

- Async: Axios (Async/Await)

# 📱 주요 화면 및 기능 명세

## 1. 메인 화면 (레시피 검색)

  - 사용자가 냉장고 속 재료를 입력하여 만들 수 있는 요리를 찾는 핵심 화면입니다.

### 🔍 재료 기반 검색

- 사용자가 입력한 재료(예: 김치, 계란)가 포함된 레시피를 검색합니다.
- 검색 결과는 Context API에 저장되어, 상세 페이지 이동 후 뒤로가기를 해도 검색 상태가 유지됩니다.

### 🎛️ 필터링 옵션

- 요리 종류 필터: 전체 / 반찬 / 국&찌개 / 일품 / 후식 등 카테고리별로 필터링하여 볼 수 있습니다. (Select 활용)

- 정확도 필터 (토글): 재료명이 정확히 일치하는 요리만 보거나, 해당 단어를 포함하는 모든 요리(예: '마' 검색 시 '마늘' 포함)를 볼 수 있도록 토글 스위치로 제어합니다. (useMemo 활용 최적화)

### ⚠️ 알레르기 정보 제공

- 검색된 요리의 재료를 분석하여 알레르기 유발 물질(예: 새우, 계란, 밀가루 등)이 포함된 경우 경고 배지를 표시합니다.

- 배지에 마우스를 올리면(Hover), 포함된 알레르기 유발 재료 목록이 툴팁으로 나타납니다.

### 👀 재료 미리보기 (Hover)

- 카드 리스트의 재료 부분에 마우스를 올리면, 잘린 전체 재료 목록을 팝업(Tooltip)으로 확인할 수 있습니다.

# 2. 검색 성공 화면 (결과 및 상세)

- 검색된 레시피를 확인하고 상세 조리법을 배우는 화면입니다.

## 2-1. 검색 결과 리스트

- 카드형 UI: 요리 사진, 이름, 분류(반찬/국 등)를 한눈에 볼 수 있습니다.

- 조리법 보기: 클릭 시 상세 조리 페이지로 이동합니다.

- 저장: 클릭 시 '나의 레시피북'에 즉시 추가됩니다. (중복 저장 방지 기능 포함)

## 2-2. 조리법 상세 보기 화면 (/recipe/view)

- 상세 정보: 완성된 요리 사진, 조리 방법(굽기, 끓이기 등), 상세 재료 목록을 제공합니다.

- Step-by-Step 조리법: 1번부터 순서대로 조리 과정과 각 단계별 이미지를 제공하여 쉽게 따라 할 수 있습니다.

- 저장 기능: 조리법을 보면서 마음에 들면 바로 '나의 레시피북'에 저장할 수 있습니다.

- 뒤로 가기: 버튼 클릭 시 이전 검색 결과 화면으로 돌아가며, 검색어와 필터 상태가 그대로 유지됩니다.

## 3. 나의 레시피북 (CRUD 관리)

- 📂 레시피 조회 (Read): 저장된 레시피 목록을 카드 형태로 조회합니다.

- 📝 메모 기능 (Update): 각 레시피에 나만의 꿀팁이나 필요한 추가 재료를 메모하고 수정할 수 있습니다.

- 🗑️ 삭제 기능 (Delete): 더 이상 필요 없는 레시피를 목록에서 영구적으로 삭제할 수 있습니다.

- 연동: 여기서도 제목이나 이미지를 클릭하면 상세 조리법 화면으로 이동합니다.

## 4. Footer (하단 정보)

- 출처 명시: 공공데이터 활용 규정에 따라 레시피 데이터의 출처를 명시합니다.

- 바로가기 링크: 버튼 클릭 시 식품안전나라 데이터셋 페이지로 이동합니다.

- 레이아웃: Flexbox를 사용하여 콘텐츠 양과 관계없이 항상 화면 최하단에 고정됩니다 (Sticky Footer).

## 🛠️ 기술적 특징 (Technical Highlights)

- SPA 아키텍처: React Router를 사용하여 페이지 새로고침 없는 부드러운 화면 전환을 구현했습니다.

- 성능 최적화: useMemo를 사용하여 필터링 로직 수행 시 불필요한 재연산을 방지하고, useCallback으로 함수 재생성을 제어했습니다.

- 전역 상태 관리: Context API와 useReducer를 도입하여 검색 상태와 레시피북 데이터를 효율적으로 중앙 관리하고 Props Drilling을 방지했습니다.

- 사용자 편의성: useRef를 사용하여 페이지 로드 시 검색창에 자동으로 포커스를 주어 입력 편의성을 높였습니다.
